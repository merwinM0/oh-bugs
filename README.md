# oh！bugs！ — 终端错误彩蛋工具（Emoji 动画版）

## 1. 项目概述

 Rust 编写的终端辅助工具。用户输入 `obugs` 命令后进入**增强终端环境**，所有原生命令正常执行，但当命令输出中出现 `error`（可配置）关键字时，终端屏幕上会随机出现飘移的虫子 Emoji（🦟 🪰 🐝 🕷️ 🦗），持续数秒后消失，不影响任何命令的正常输入输出。

## 2. 核心功能

### 2.1 进入与退出

| 操作               | 效果                     |
| ------------------ | ------------------------ |
| `obugs`            | 启动 BugFlare 会话       |
| `exit` 或 `Ctrl+D` | 退出，恢复正常终端       |
| `Ctrl+C`           | 取消当前命令，不退出环境 |

### 2.2 工作流程

用户输入命令 → 程序转发给真实 Shell → 捕获输出流
                                              ↓
                                    实时匹配 "error"
                                              ↓
                                    触发虫子动画（异步）
                                              ↓
                                    原样显示命令输出

### 2.3 虫子动画规则

| 属性     | 规则                                                  |
| -------- | ----------------------------------------------------- |
| 虫子池   | 🦟（蚊子）、🪰（苍蝇）、🐝（蜜蜂）、🕷️（蜘蛛）、🦗（蟋蟀） |
| 触发条件 | 每匹配到 1 个 "error" 关键字                          |
| 数量     | 每次触发随机生成 **2~5 只**虫子                       |
| 出现位置 | 终端可见区域的随机列位置（行固定在当前视口上半区）    |
| 移动方式 | 每 100ms 向右/左漂移 1~2 列，同时随机上下浮动 0~1 行  |
| 生命周期 | 单只虫子存活 **2~3 秒**后消失                         |
| 最大并发 | 屏幕同时最多显示 30 只虫子（防止过度占用）            |

## 3. 技术选型

### 3.1 编程语言：Rust

- 无需运行时，单二进制文件分发
- 极低的内存占用（动画部分 < 10MB）
- 对终端 I/O 和并发的原生支持

### 3.2 核心依赖

| Crate                  | 用途                                       |
| ---------------------- | ------------------------------------------ |
| `crossterm`            | 终端控制（光标移动、颜色、清屏、捕获事件） |
| `tokio` 或 `async-std` | 异步处理输出流 + 动画定时器                |
| `regex`                | 匹配错误关键字                             |
| `signal-hook`          | 处理 Ctrl+C 等信号                         |
| `anyhow`               | 错误处理                                   |

## 4. 架构设计

### 4.1 模块划分

src/
├── main.rs          # 入口，初始化与事件循环
├── shell.rs         # PTY 代理，子进程管理
├── watcher.rs       # 输出解析 + 错误检测
├── animation.rs     # 虫子动画引擎（核心）
├── terminal.rs      # crossterm 封装
└── config.rs        # 配置加载（关键字、虫子数量、存活时间等）
### 4.2 动画引擎设计要点

- **全局状态**：维护一个 `Vec<Bug>`，每只虫子包含：
  - `x, y`: 当前列、行坐标
  - `emoji`: 5 种 Emoji 之一
  - `birth`: 创建时的 Instant
  - `direction`: -1（向左）、1（向右）
- **刷新策略**：独立 spawn 一个异步任务，每 80~120ms：
  1. 清除上一帧所有虫子位置（用空格覆盖）
  2. 更新每只虫子的坐标
  3. 超时虫子从 Vec 中移除
  4. 重新绘制所有虫子
- **与输入互不干扰**：动画任务和输出捕获任务通过 `mpsc::channel` 通信，动画不会阻塞用户输入

### 4.3 关键挑战与解决方案

| 挑战               | 解决方案                                                 |
| ------------------ | -------------------------------------------------------- |
| 用户输入时动画闪烁 | 动画帧仅在无键盘输入时刷新；或使用双缓冲机制             |
| 命令输出覆盖虫子   | 检测到新输出时，先清空虫子区域，输出完成后恢复动画       |
| 窗口大小变化       | 监听 SIGWINCH，重新获取终端尺寸，超出边界的虫子自动移除  |
| 性能（低配终端）   | 可配置刷新间隔；默认启用“省电模式”（输入时降低动画帧率） |

## 5. 安装与使用

### 5.1 编译
```bash
cargo build --release
# 二进制文件位于 target/release/obugs
```

### 5.2 运行
```bash
# 直接启动
./obugs

# 或安装到 PATH
cargo install --path .
obugs

# 或直接在.local/bin下放入obugs
chmod  +x obugs
```

### 5.3 配置文件 `~/.config/obugs/config.toml`
```toml
error_keywords = ["error", "fail", "exception", "fatal"]  # 大小写不敏感
min_bugs = 2
max_bugs = 5
bug_lifetime_ms = 2500
refresh_rate_ms = 100
max_concurrent_bugs = 30
```

## 6. 已知限制与未来扩展(暂时不做)

### 6.1 当前限制
- Emoji 在某些老旧终端（如 Windows 旧版 cmd）可能显示为方块
- 多行命令（如 `python` 交互式）不支持（因为无法区分输出和提示符）
- 不能用于已经运行的长期进程（如 `top`），因为会破坏其界面

### 6.2 未来可扩展功能
- **撞击效果**：虫子碰到屏幕边缘会反弹
- **用户交互**：按 `Ctrl+B` 手动放出一波虫子
- **录音模式**：匹配到 error 时播放“嗡嗡”声（需额外音频库）

本方案不依赖任何图形库，完全基于标准终端的字符输出和 Emoji，保证了：
- ✅ 在 SSH、tmux、任何现代终端中完美运行
- ✅ 极低资源占用（适合笔记本/服务器）
- ✅ 代码简洁（核心动画引擎 < 200 行）
- ✅ 符合“虫子乱飞”的趣味需求
